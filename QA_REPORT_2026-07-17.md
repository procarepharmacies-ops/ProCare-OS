## 5. Bugs found & fixed (this session)

### BUG-1 — eStock mirror crashed on `decimal.Decimal` (0% mirroring in dev/CI)
- **File:** `src/backend/app/services/etl.py`
- **Root cause:** `_load_products` inserted `unit_big`/`unit_small` as **raw `r.get(...)`** values. On SQL Server these arrive as `decimal.Decimal`, which MSSQL binds natively — so production never failed. On **SQLite** (dev/CI), SQLAlchemy cannot bind `Decimal` into the `String(50)` columns → `type 'decimal.Decimal' is not supported` → the *entire* products load aborted → **0 rows mirrored**.
- **Fix:** added a `_str()` coercer (Decimal/any → `str`, None-safe) and applied it to `unit_big`/`unit_small` in both the create and update paths. Preserves the unit name; makes the mirror dialect-agnostic.

### BUG-2 — SQLite "database is locked" on concurrent access
- **File:** `src/backend/app/db/base.py`
- **Root cause:** default SQLite (rollback-journal, single-writer) deadlocks when the lifespan seeding session, the background sync thread and concurrent HTTP requests share the file.
- **Fix:** on SQLite, the connect event now also sets `PRAGMA journal_mode=WAL` and `PRAGMA busy_timeout=5000`, so writers don't block readers and lock waits time out gracefully.

### BUG-3 — Repo would commit 154 stray `.cab` (DirectX) files
- **File:** `.gitignore`
- **Root cause:** missing `*.cab` / DirectX-redist patterns; `git status` showed 154 untracked `.cab` files that would have been committed.
- **Fix:** added `*.cab`, `APR2007_*/`, `DEC2006_*/`, `vcredist_*.exe`, `*.msi` to `.gitignore`.

No credential files were touched or committed (`config/connections.json` is git-ignored).
